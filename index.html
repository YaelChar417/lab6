Preguntas y fuentes

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

En primera es importante porque garantizan que se envían coincidan con los requisitos de información, del mismo modo, garantizan una mejor experiencia de usuario, ya que si corregimos los datos del lado del cliente este los corrige rapidamente, caso contrario a que si mandaramos esos datos al servidor sin verificar probablemente se rechazaría y eso produciría un retraso en la comunicación.

Validación de formularios de datos - Aprende desarrollo web | MDN. (s. f.). MDN Web Docs. https://developer.mozilla.org/es/docs/Learn_web_development/Extensions/Forms/Form_validation

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

La forma más sencilla de hacerlo es desactivando el javascript de la pagina web a la que se esta accediendo, otra forma que igual vimos en clase es cambiar el valor a las variables usando la consola de la pagina web.

Ada. (2020, 12 septiembre). Cómo saltarse la validación HTML5 de un formulario. https://didacticode.com/como-saltarse-validacion-html5-formulario/

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque hacerlo mejora la experiencia que tiene el usuario y aunque es cierto que se puede saltar, hacerlo garantiza que incluso para los usuarios que no tengan estas intenciones sus datos se manden de forma deseable, pero incluso si se deseará evitarlo lo mejor es también validar con la parte de Backend

Ciordia, N. G. (2021, 16 diciembre). Validación Básica de Formularios en JavaScript. freeCodeCamp.org. https://www.freecodecamp.org/espanol/news/validacion-basica-de-formularios-en-javascript/
